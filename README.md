## 1. Περιγραφή της Εφαρμογής
Η παρούσα εφαρμογή αναπτύχθηκε σε γλώσσα προγραμματισμού Java με σκοπό την πλήρη ψηφιακή διαχείριση ενός σύγχρονου ταξιδιωτικού πρακτορείου. Το σύστημα επιτρέπει τη διαχείριση τουριστικών πακέτων, πελατών και πόλεων, προσφέροντας παράλληλα δυναμικό υπολογισμό κοστών, τιμών πώλησης και αναλυτικών κερδών ανά πακέτο μέσω ενός διαδραστικού περιβάλλοντος κονσόλας (CLI).

---

## 2. Αναλυτική Παρουσίαση Κλάσεων & Αρχών OOP
Η εφαρμογή στηρίζεται αποκλειστικά στο αντικειμενοστραφές μοντέλο προγραμματισμού (OOP) και καλύπτει πλήρως την ύλη του μαθήματος:

* **City**: Αντιπροσωπεύει μια πόλη. Τροποποιήθηκε ώστε να υποστηρίζει μόνιμο αναγνωριστικό (`id` INT) συνδεδεμένο άμεσα με τη MySQL.
* **Travel (Abstract)**: Αφηρημένη κλάση που λειτουργεί ως βάση για τις μετακινήσεις, περιλαμβάνοντας τις πόλεις σύνδεσης και το κόστος.
* **AirFlight, Boat, Train, Bus**: Υποκλάσεις που κληρονομούν την κλάση Travel και υλοποιούν **πολυμορφικά** τη μέθοδο εκτύπωσης `print()`.
* **PackageTour**: Υλοποιεί την αρχή της **σύνθεσης (Composition)**, καθώς περιέχει dynamic λίστες αντικειμένων City και Travel. Υπολογίζει δυναμικά το συνολικό κόστος και την τελική τιμή.
* **Customer**: Αντιπροσωπεύει τον πελάτη και περιέχει μια συλλογή (ArrayList) με τα τουριστικά πακέτα που έχει αγοράσει (**συσχέτιση αντικειμένων**).

### 💾 Αρχιτεκτονική Υβριδικής Αποθήκευσης (Data Persistence)
* **DBManager (MySQL - JDBC):** Υπεύθυνη για τη live σύνδεση με τη σχεσιακή βάση και την εκτέλεση CRUD λειτουργιών για τις Πόλεις (με υποστήριξη IDs), τους Πελάτες, καθώς και για την αυτόματη καταγραφή των βασικών στοιχείων των Πακέτων στον νέο πίνακα `packages`.
* **FileManager (Advanced Java I/O):** Υλοποιεί το **Serialization** για την εγγενή αποθήκευση και ανάκτηση ολόκληρων των Object Graphs (σύνθετα πακέτα, πολυμορφικά ταξίδια, ιστορικό αγορών πελατών) στο δυαδικό αρχείο `agency_data.dat`.

## 3. Πρόσφατες Αρχιτεκτονικές Αναβαθμίσεις
Κατά το τελικό στάδιο ανάπτυξης, το σύστημα βελτιστοποιήθηκε με τις εξής λειτουργίες:
1.  **Dynamic Package Generation (Μενού 1):** Η δημιουργία πακέτου έγινε πλήρως δυναμική. Το σύστημα τραβάει live τις διαθέσιμες πόλεις από τη MySQL και ο χρήστης τις εισάγει στη διαδρομή χρησιμοποιώντας αποκλειστικά το ID τους.
2.  **Interactive Cost & Profit Analysis (Μενού 5):** Αντί για ένα ξερό άθροισμα, το μενού οικονομικών στοιχείων πλέον προβάλλει dynamic λίστα πακέτων. Ο χρήστης επιλέγει index και το σύστημα εκτελεί ανάλυση (Καθαρό Κόστος, Καθαρό Κέρδος Γραφείου 25%, Τελική Τιμή Πώλησης).
3.  **Μηχανισμός Διαγραφής Πακέτων (Delete):** Ενσωματώθηκε επιλογή πλήρους διαγραφής καταχωρημένων πακέτων από τη runtime μνήμη με sequential scan αναζήτηση βάσει ID.
4.  **Greeklish Encoding & Code Comments:** Όλα τα runtime μηνύματα της κονσόλας και τα σχόλια τεκμηρίωσης μετατράπηκαν σε Greeklish, εξασφαλίζοντας την ομαλή εκτέλεση της εφαρμογής χωρίς προβλήματα ασυμβατότητας χαρακτήρων (`??`) σε οποιοδήποτε περιβάλλον εργαστηρίου.

## 4. Οδηγίες Εγκατάστασης και Εκτέλεσης (Lab Setup)
1.  Ενεργοποιήστε τα **Apache** και **MySQL** services μέσα από το XAMPP Control Panel.
2.  Εισάγετε τη βάση δεδομένων `travel_agency_db` στο phpMyAdmin. Επιβεβαιώστε την ύπαρξη των 3 πινάκων (`cities`, `customers`, `packages`).
3.  Ανοίξτε το project στο Apache NetBeans.
4.  Κάντε δεξί κλικ στο φάκελο **Libraries** -> **Add Library** -> Επιλέξτε **MySQL JDBC Driver** για την ενεργοποίηση του Connector.
5.  Εκτελέστε **Clean and Build** και στη συνέχεια **Run**.

## 5. Οδηγίες Επίδειξης (Demo) & Διαχείριση Εξαιρέσεων
* **Μηχανισμός Demo:** Κατά την πρώτη εκκίνηση, αν οι λίστες είναι κενές, η εφαρμογή παράγει αυτόματα ένα ολοκληρωμένο περιβάλλον επίδειξης (Πόλεις με IDs, σύνθετο πολυμορφικό πακέτο "Europe Panorama" και έναν έτοιμο πελάτη).
* **Exception Handling:** Έχει ενσωματωθεί πλήρης διαχείριση σφαλμάτων με try-catch blocks σε κάθε επίπεδο, θωρακίζοντας την εφαρμογή από λανθασμένες εισαγωγές χρηστών (`NumberFormatException`) ή αποτυχίες επικοινωνίας με τη βάση δεδομένων (`SQLException`).
